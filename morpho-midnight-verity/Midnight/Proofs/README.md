`ContractShape.lean` pins the generated Verity model bodies for these functions
with `rfl` examples, so formula drift is caught by `lake build Midnight.Proofs`.
The general `normalModeRcfLocalSequence` keeps the RCF threshold branch. The
`normalModeRepaidInputLocalSequence` and `normalModeLiquidateLocalSequence`
surfaces specialize to `repaidUnits = maxRepaid`, where the guard's left
disjunct is already satisfied, so they expose no dead `rcfThreshold` parameter.
The current proof/extraction boundary is summarized in `TRUST_BOUNDARIES.md`.
The current completion audit is summarized in `COMPLETION_AUDIT.md`.

## RCF Health Recovery

File: `RCF.lean`
Boundary files: `CollateralLoop.lean`, `Refinement.lean`
Storage projection: `Storage.lean`

Solidity anchor: `midnight/src/Midnight.sol` lines 655-667.

The model keeps the same normal-mode `maxRepaid` expression:

```lean
mulDivUp (debt - maxDebt) (WAD * WAD) (WAD * WAD - lif * lltv)
```

The branch condition matches Solidity: the formula is used when `lltv < WAD`,
and the helper returns `type(uint256).max` when `lltv >= WAD`. The recovery
proofs additionally require `lif * lltv < WAD^2`, which is the
positive-denominator domain for the natural-number arithmetic.

The proportional-slash theorem proves the core accounting fact: if stored lender
credits are covered by `totalUnits`, and both lender credits and total units are
slashed by the same loss ratio, the post-slash lender-credit sum is covered by
the post-slash `totalUnits`.

The loss-factor-step theorem specializes this to `updatePositionView`'s
loss-factor ratio: lenders synced to the old market loss factor are covered after
the market moves to a new loss factor, provided the new `totalUnits` covers the
old `totalUnits` multiplied by the same `(UINT128_MAX - newLossFactor) /
(UINT128_MAX - oldLossFactor)` ratio.

The bad-debt lemma proves that Midnight's exact loss-factor update:

```solidity

totalUnits -= badDebt;
```

satisfies that total-units ratio bound.
